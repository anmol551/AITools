1 Research Design: Financial Risk Prediction Pipeline

The research design for financial risk prediction in loan approval begins with a literature survey on machine learning techniques for credit scoring, followed by data collection from a synthetic dataset. The pipeline includes dataset exploration to understand data characteristics, preprocessing steps such as encoding categorical variables and feature selection to identify key predictors. Subsequent stages involve feature engineering to create new variables, scaling of features and train-test splitting to prepare data for modelling. Three regression models are then built and evaluated for predicting continuous risk scores with subsequent classification conversion. The process extends to integrating a large language model for decision explanation and deploying a web application for real-time predictions.

This design ensures a comprehensive approach from data handling to deployment, incorporating domain-specific adaptations like threshold-based classification and business rule application. Model comparison aids in selecting effective architectures while the web app provides user interaction for input features, risk scoring and automated reports via an API call, enhancing practical utility in financial decision-making.

11 Model Building: Regression Models for Risk Prediction

Model building employs various architectures to predict financial risk scores.

ElasticNet Regressor
ElasticNet regressor combines lasso and ridge penalties for feature selection and regularisation. It was chosen for this research due to its ability to handle multicollinearity in financial datasets, making it suitable for mixed feature types. This model balances sparsity and smoothness, aiding in robust predictions for risk scores.

The model operates by minimising a loss function that includes both L1 and L2 norms, with parameters like alpha and l1 ratio tuned for optimal fit. [EQ3: ElasticNet loss function] Sarkar et al. (2024) illustrate its effectiveness in handling correlated predictors.

CNN Regressor Architecture
The CNN regressor uses convolutional layers for feature extraction in tabular data. It was selected for its capacity to capture sequential patterns in financial features, improving upon linear models.

This model processes inputs through layers that detect local dependencies, with parameters set for depth and activation.

Conv1D Layer
The Conv1D layer applies filters to input sequences for feature detection. In this architecture, it extracts relevant patterns from financial time-series-like data, contributing to better risk assessment.

Internally, it performs sliding window operations on inputs, with parameters such as 32 filters and kernel size of 2. [EQ4: Convolution operation formula] Gupta and Sharma (2026) explain its utility in tabular learning.

BatchNormalisation Layer
BatchNormalisation stabilises the training process by normalising activations. It plays a role in this model by reducing internal covariate shift, thus speeding up convergence for financial predictions.

The layer computes mean and variance of activations and adjusts them, using parameters to maintain stability. This ensures efficient learning without altering the overall architecture.

Flatten Layer
The Flatten layer converts multi-dimensional outputs to a vector for further processing. In this setup, it bridges convolutional and dense layers, enabling seamless integration in the risk model.

Operationally, it reshapes data by flattening arrays, preparing them for fully connected layers with specific input dimensions.

Dense Layer
The Dense layer forms fully connected networks for final predictions. It integrates features from prior layers, finalising the output in this architecture.

The layer applies linear transformations followed by activation, with 64 units and ReLU function, allowing complex mappings in risk scoring.

Dropout Layer
Dropout prevents overfitting by randomly dropping units during training. It enhances generalisation in this model by regularising the network against financial data variances.

During operation, it sets a dropout rate to zero out connections, with parameters ensuring balanced training.

TabNet Regressor
TabNet regressor uses attention mechanisms for interpretable tabular learning. It was picked for its focus on feature importance in financial contexts, providing transparency in risk predictions.

The model works through sequential processing with attention blocks, parameters like step count and learning rate tuned for efficiency. [EQ5: Attention mechanism formula] Arik and Pfister (2021) demonstrate its advantages in decision-making.

Attention Mechanism
The Attention Mechanism in TabNet selects relevant features dynamically. It contributes by focusing on important inputs for each decision step in the model.

Internally, it computes feature masks using parameters such as gamma, allowing adaptive learning in risk prediction.

12 Model Performance Evaluation Metrics

Evaluation metrics assess the accuracy of financial risk models.

Metrics like mean absolute error measure the average prediction difference, providing insight into error magnitude for risk scores. Root mean square error quantifies larger errors more heavily, indicating variability in predictions relevant to loan decisions.

R-squared evaluates how well the model explains data variance, crucial for understanding prediction reliability in financial contexts. Accuracy reflects correct classifications from thresholded scores, essential for binary outcomes. Precision, recall and F1-score balance true positives and false alarms, vital for fairness in credit approvals. ROC-AUC measures discrimination ability, aiding in threshold selection for risk assessment.

13 LLM Integration: OpenAI GPT-4o-mini in Risk Pipeline

LLM integration incorporates a generative language model to enhance explanations. OpenAI GPT-4o-mini was chosen for its efficiency and ability to process structured prompts, making it ideal for financial report generation.

In the pipeline, it generates detailed underwriting reports post-prediction, using an API for input-output handling. This improves decision transparency by providing interpretable insights. Golec and AlabdulJalil (2025) explore how LLMs aid in explainable AI for credit systems.

14 Web Application: Streamlit for Risk Deployment

Web application development uses a framework for interactive interfaces. Streamlit was selected for its simplicity and rapid deployment features, suitable for financial tools.

The app processes user inputs through feature engineering and model prediction, outputting risk scores and reports via API calls. This setup offers user-friendly access to predictions, enhancing practical application in lending. Ben Jebli and Zoghlami (2025) discuss the benefits of such frameworks in real-time systems.
