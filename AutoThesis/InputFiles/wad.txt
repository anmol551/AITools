Web App Development:
The development of a web application for detecting AI-generated text focuses on practical implementation using Dash as the core framework, paired with dash_bootstrap_components for a polished look. Users interact through a clean interface built with dbc.Textarea, allowing text entry and submission via a styled button for analysis. The process begins with text cleaning through functions like basic_clean_text, which handles case conversion and punctuation removal. Another function, advance_clean_text, deals with expanding contractions and transforming numbers into words while clearing emojis. The third step, stop_lemma_clean_text, breaks text into tokens, removes common words, and applies lemmatization for consistency. After cleaning, the text is transformed into embeddings using SentenceTransformer with all-MiniLM-L6-v2 for semantic representation. A pre-trained Random Forest model, accessed as rf_model, then classifies the input as human-written or AI-generated. Results are presented with confidence scores, often shown as a percentage like 85%, alongside a visual bar for clarity. This setup ensures smooth user experience and accurate text classification through a streamlined pipeline of processing and prediction steps.
