**Summary of AI vs. Human Text Detection Code**

The Python code implements a machine learning pipeline for detecting AI-generated versus human-generated text, leveraging deep learning and context-aware techniques. Its primary purpose is to classify text from a dataset of 487,235 records into human (62.8%) or AI-generated (37.2%) categories, using a comprehensive approach that includes data preprocessing, feature extraction, model training, and evaluation. The code employs the DistilBERT transformer model for high-accuracy text classification, supplemented by other models like FocusCNN, AttentionCNN, and Context RF. It also includes extensive text analysis and visualization to explore linguistic differences, culminating in a robust evaluation framework with metrics such as accuracy, precision, recall, F1 score, and Hamming loss.

Key components include data loading from a CSV file (`AI_Human.csv`), preprocessing with libraries like `nltk` and `contractions` for text cleaning (e.g., removing punctuation, stop words), and feature extraction through text analysis (e.g., `text_length`, `word_count`, `num_sentences`). The core logic utilizes the `SentenceTransformer` for embeddings and `DistilBertForSequenceClassification` for classification, with a custom `AITextDataset` class to handle tokenization and data loading via `DataLoader`. The execution flow starts with data loading and exploratory analysis (e.g., word clouds, bigram/trigram plots), followed by dataset splitting (60% train, 20% validation, 20% test), model training with `AdamW` optimizer, and evaluation using metrics like `accuracy_score` and `roc_auc_score`. Visualizations, including confusion matrices and ROC curves, are generated using `seaborn` and `matplotlib`.

The training process involves one epoch with a batch size of 32 and a learning rate of 2e-5, optimized by a linear scheduler. The DistilBERT model achieves superior performance with 98.12% accuracy, 98.13% precision, 98.12% recall, 98.12% F1 score, and 1.87% Hamming loss, outperforming FocusCNN (97.70% accuracy), Context RF (97.05%), and AttentionCNN (94.38%). Challenges include resource constraints, limiting the dataset to 10,000 samples for DistilBERT and excluding LSTM/GRU models due to computational demands. No web application elements are implemented, but results are saved to a CSV file (`model_evaluation_results.csv`) for further analysis, emphasizing the code’s focus on model development and evaluation.
