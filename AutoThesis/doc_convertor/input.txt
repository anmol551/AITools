ABSTRACT

Loan approval decisions in the financial sector often suffer from inaccuracies due to complex risk factors such as credit history and income levels which lead to high denial rates and potential losses for lenders. Machine learning algorithms help address this by predicting loan risk scores and approval outcomes through data-driven pattern analysis. The selected dataset is the synthetic financial risk dataset for loan approval sourced from Kaggle. The methodology includes preprocessing steps like dropping irrelevant columns encoding categorical variables with OneHotEncoder and LabelEncoder transforming ordinal features engineering new variables using Recursive Feature Elimination and scaling numerical features with StandardScaler. Models trained include ElasticNet CNN and TabNet. The best model TabNet achieved an accuracy score of 89.1%. This model surpassed the dataset's inherent class imbalance benchmark as evidenced by improved performance metrics compared to baseline expectations. The web application developed uses the Streamlit framework to provide an interactive platform for users to input financial data receive risk predictions via the TabNet model and view integrated reports. This setup enables real-time decision support for loan governance. The project focused on enhancing model interpretability through integrated explanations generated by large language models to aid users in understanding predictions. (230 words)

4.3. Data Preprocessing for Loan Features

Data preprocessing prepared the loan features for modelling by handling categorical and non-informative data.

One-Hot Encoding transformed specific categorical columns using OneHotEncoder. The process applied to EmploymentStatus HomeOwnershipStatus MaritalStatus and LoanPurpose with sparseoutput set to False to produce a dense array. This step created binary indicator features for each category replacing the original text values and expanding the feature set for better model compatibility.

Label Encoding of Education Level

Label Encoding converted the EducationLevel column using LabelEncoder. The encoder assigned integers to each unique category based on default parameters resulting in a single numerical column. This ordinal transformation allowed the feature to integrate into regression models without further adjustments.

Feature selection identified the most relevant features using Recursive Feature Elimination with RandomForestRegressor. The implementation involved RFE from scikit-learn with estimator set to RandomForestRegressor nfeaturestoselect at 10 and default other parameters to rank and select features based on importance scores. This reduced the feature space to ten key variables including AnnualIncome and RiskScore.

The process achieved a subset of features that minimized multicollinearity and improved efficiency. A correlation heatmap visualized relationships among the selected features.

5.6. RESEARCH NOVELTY

This research presents several key innovations in financial risk modelling. It introduces a unified approach that combines continuous risk score prediction and binary loan approval decisions into one predictive system. This method uses a single TabNet regressor where the output is thresholded at 50% to generate decisions ensuring consistency between risk assessment and approval outcomes.

The study employs TabNet's attention mechanism with entmax sparse activation and five sequential steps to offer feature-level interpretability. This design suits regulated financial contexts by providing transparency that standard neural networks lack. It also incorporates a virtual batch size of 128 within larger batches of 512 to enhance generalisation on tabular data through ghost batch normalisation avoiding common training instabilities.

Another contribution involves creating six composite financial indicators such as the Debt Stress Index calculated as DebtToIncomeRatio times InterestRate and the Composite Risk Index as a weighted mix of credit score deficit debt ratio interest rate and risk flags. These indicators integrate domain knowledge into the feature set reducing the model's reliance on raw data.

The system adds a governance layer that applies lending policy rules as overrides on predictions above 50%. This two-tier setup combines model inference with rule-based enforcement to align with operational needs.

Finally the integration of GPT-4o-mini transforms numerical outputs into structured six-section underwriting reports. This feature creates natural language justifications that aid audits reviews and communication fostering human-AI teamwork in decision processes.
