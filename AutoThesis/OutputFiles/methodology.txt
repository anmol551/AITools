### Methodology

#### 1. Research Design for Financial Risk Prediction

This research outlines a detailed pipeline for assessing financial risk in loan approval scenarios using a synthetic dataset. The process begins with data collection from an online repository followed by thorough exploration to understand the dataset's structure and patterns. Key stages include cleaning and transforming data, selecting relevant attributes, and creating new variables to improve model performance. The pipeline then moves to building predictive models, evaluating their outputs, and comparing their effectiveness for risk score estimation and loan approval decisions. Three distinct approaches are explored: a linear regression method, a convolutional neural network, and an attention-based tabular learning model.

Subsequent steps involve integrating an advanced language model for generating detailed reports and developing a web-based platform named Streamlit for real-time risk assessment. The pipeline concludes with testing the application across various applicant profiles to ensure practical usability. This comprehensive design aims to balance technical precision with actionable decision-making in financial contexts.

#### 2. Dataset Description for Loan Risk Analysis

The dataset used in this study is a synthetic collection focused on financial risk for loan approval, sourced from Kaggle (https://www.kaggle.com/datasets/lorenzozoppelletto/financial-risk-for-loan-approval/data). It contains 20,000 records with 37 attributes covering diverse applicant details. This large volume of data provides a solid foundation for building reliable predictive models.

The choice of this dataset stems from its comprehensive nature, capturing a wide range of financial and personal factors relevant to loan risk assessment. Its synthetic origin allows for experimentation without privacy concerns, making it suitable for academic exploration.

Ethically, using artificial data avoids risks associated with real personal information. It also ensures that biases in real-world data are not replicated during model training. The dataset shows a notable imbalance in approval outcomes, which poses challenges for analysis. Key representations include numerical indicators of financial health, categorical descriptors of personal status, and target variables for both continuous risk scores and binary approval decisions.

#### 9. Model Building for Risk Score Prediction

This section outlines the development of predictive models for financial risk assessment.

##### 9.1 ElasticNet Regression Model

ElasticNet regression is a linear approach combining two regularisation techniques to predict continuous risk scores. This model was selected for loan risk analysis due to its ability to handle high-dimensional data while preventing overfitting. Its simplicity makes it an ideal baseline for comparison with more complex architectures. In financial contexts, where interpretability matters, this method offers clarity in understanding variable impacts on risk outcomes.

The model operates by balancing penalty terms to control complexity, with parameters set for maximum iterations and a fixed seed for consistency. A search method optimised its hyperparameters to improve performance. [EQ_14: regularisation balance expression]; [EQ_15: objective minimisation function]. According to Mazur (2025), such linear models provide a strong foundation for financial predictions due to their straightforward mechanics. This approach ensures a reliable starting point for risk estimation.

##### 9.2 Convolutional Neural Network (CNN) Regressor

A 1D Convolutional Neural Network was chosen for its strength in capturing local patterns within sequential or structured data. In loan risk prediction, this model excels at identifying subtle relationships among financial variables over traditional methods. Its layered structure suits the complexity of risk score estimation, making it a valuable tool for deeper analysis in this domain.

The architecture processes input through multiple stages, with specific settings for filters, kernel sizes, and activation functions. Training parameters include a defined number of cycles and batch sizes. [EQ_16: convolution operation]; [EQ_17: loss optimisation expression]. As per Akyüz and Güler (2025), convolutional networks can uncover hidden patterns in financial datasets effectively. This model adds depth to the predictive framework used in this study.

###### 9.2.1 Conv1D Layer

The Conv1D layer is a core component of the CNN architecture, designed to extract local features from input data. In the context of loan risk prediction, it helps detect patterns across financial attributes that might indicate risk levels. This layer focuses on small segments of data to build a hierarchical understanding.

It operates by sliding a filter over the input to compute feature maps, with defined filter counts and kernel sizes for each instance. The activation function introduces non-linearity to capture complex relationships. [EQ_18: feature map computation]. This mechanism ensures that relevant financial indicators are prioritised, contributing to accurate risk score predictions within the broader network structure.

###### 9.2.2 Batch Normalisation Layer

Batch Normalisation stabilises training by normalising layer inputs across mini-batches. Within this CNN for loan risk analysis, it reduces internal covariate shift, speeding up convergence. This layer ensures consistent performance across diverse financial data inputs, enhancing model stability.

It adjusts mean and variance during training, applying scaling and shifting parameters to maintain data distribution. This process mitigates issues arising from varying data scales. [EQ_19: normalisation adjustment formula]. By maintaining a steady flow of information, this layer supports effective learning of risk patterns, ensuring the network adapts smoothly to the dataset's characteristics.

###### 9.2.3 Flatten Layer

The Flatten layer reshapes multi-dimensional data into a single vector for further processing. In this CNN for financial risk prediction, it connects convolutional outputs to dense layers, enabling comprehensive risk assessment. This transformation is crucial for integrating learned features into final predictions.

It works by collapsing spatial dimensions while preserving all extracted information, ensuring no data loss. The process is straightforward but essential for downstream compatibility. [EQ_20: dimension reshaping process]. This step ensures that complex patterns identified earlier are effectively translated into a format suitable for risk score computation within the network.

###### 9.2.4 Dense Layer

The Dense layer performs high-level reasoning by connecting all inputs to outputs via weighted computations. In loan risk modelling, it synthesises features extracted by prior layers into a cohesive risk score. This layer is key to producing meaningful predictions from processed financial data.

It applies weights and biases to input values, followed by an activation function to introduce non-linearity. Specific neuron counts define its capacity for each instance. [EQ_21: weighted sum computation]. This mechanism allows the model to weigh the importance of various financial factors, ensuring accurate risk estimation as part of the broader architecture.

###### 9.2.5 Dropout Layer

Dropout randomly deactivates a fraction of neurons during training to prevent overfitting. In this CNN for financial risk analysis, it promotes generalisation by reducing reliance on specific features. This layer ensures the model remains adaptable to unseen applicant data.

It operates by setting a dropout rate, effectively ignoring selected units in each training pass. This randomness enhances model resilience across diverse inputs. [EQ_22: random deactivation proportion]. By introducing variability, this layer helps maintain prediction reliability, ensuring the network does not overfit to training data patterns in risk assessment tasks.

##### 9.3 TabNet Regressor

TabNet is an attention-based architecture designed for tabular data, making it ideal for loan risk prediction. Its ability to focus on relevant features while maintaining interpretability aligns with the needs of financial decision-making. This model was chosen for its advanced handling of structured data, offering superior performance over traditional methods in this specific context.

It employs a sequential attention mechanism to select important variables at each step, with defined parameters for feature dimensions and sparsity. #### 10. Model Performance Evaluation Metrics

This section defines the metrics used to assess model performance in financial risk prediction. Each metric provides unique insights into prediction quality for loan approval scenarios.

Mean Absolute Error measures the average magnitude of prediction errors, reflecting accuracy in risk score estimation. It is vital for understanding how close predictions are to actual values in financial contexts. [EQ_25: average error calculation].

Mean Squared Error quantifies the average squared difference between predicted and actual scores, penalising larger errors more heavily. It highlights model consistency in high-stakes loan decisions. [EQ_26: squared error average].

Root Mean Squared Error, the square root of MSE, provides error magnitude in original units, aiding interpretation in risk assessment. It balances sensitivity to outliers in financial predictions. [EQ_27: error magnitude transformation].

R-squared indicates the proportion of variance explained by the model, showing its fit for loan risk data. It is crucial for gauging predictive power. [EQ_28: variance explanation ratio].

Explained Variance assesses how well the model accounts for data variation, critical for reliable financial outcomes. [EQ_29: variation capture measure].

Classification metrics like Accuracy, Precision, Recall, F1-Score, and ROC-AUC evaluate binary loan approval decisions derived from risk scores, ensuring balanced performance assessment. [EQ_30: classification performance measures].
