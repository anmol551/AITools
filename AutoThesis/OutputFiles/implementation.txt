#### Data Characteristics for AI vs. Human Text Detection

The dataset comprises 487235 records with varied data types including text and numerical features. Summary statistics reveal a mean word count of 393.1 and a median of 363. The mean number of sentences stands at 20.8. Average word length is 4.76 characters while punctuation marks average at 48.1 per entry. Standard deviation for word count is notable at 186.2. Minimum word count is 1 and maximum reaches 1750. Quartiles for word count are 246 at 25%, 363 at 50%, and 493 at 75%, showing the spread of text lengths across the dataset.

#### Text Data Preparation
This section details the steps taken to prepare the dataset for distinguishing AI and human text through various cleaning and transformation methods.

##### Text Cleaning
Initial cleaning involved converting all text to lowercase and removing punctuation marks to ensure uniformity. This process targeted non-word and non-space characters, replacing them with empty strings. Such steps were crucial to eliminate noise that could affect analysis. Tokenisation followed, breaking down text into individual words using standard natural language processing tools. This allowed for a granular examination of content.

Stopwords were filtered out from the tokenised data to focus on meaningful terms. This reduced the dataset size without losing critical information. The cleaned text provided a clearer base for subsequent feature creation and model training, ensuring that irrelevant elements did not skew the results. These efforts streamlined the data for effective pattern recognition.

##### Data Type Conversion
Ensuring consistent data types was a key step in preparation. The target column indicating text origin was converted to an integer format to align with model requirements. This transformation was applied across all records to avoid mismatches during processing. Such consistency proved essential for seamless integration with analytical tools.

This conversion eliminated potential errors in data handling, maintaining integrity throughout the pipeline. Each record was checked to confirm the correct format, supporting accurate categorisation. By standardising data types, the groundwork was laid for reliable feature extraction and model input, contributing to the overall quality of the analysis.

##### Feature Creation and Selection
Feature creation enriched the dataset by quantifying textual attributes to aid in distinguishing between AI and human content. Several metrics were derived from the raw text, including text length as total characters, word count as the number of words, and sentence count by tallying specific punctuation marks. Average word length was computed by dividing total characters by word count, defaulting to 1 if no words were present. Unique word count captured distinct terms after converting to lowercase.

Further features included counting fully uppercase words, digit characters, and punctuation marks per entry. These eight engineered features, alongside original text and origin labels, expanded the dataset's depth. This process, applied across all 487235 records, aimed to capture stylistic and structural differences. By quantifying such traits, the dataset became more informative for models to detect subtle variations. Each feature was calculated systematically, ensuring no data was overlooked. These additions provided a comprehensive view, supporting better differentiation during model training.

#### Model Development for Text Classification
This section introduces the approach to building models for classifying text as AI or human-generated.

##### DistilBERT Implementation
DistilBERT was trained as a primary classifier for text detection. This transformer-based model used a pre-trained base named distilbert-base-uncased with settings adjusted for two output labels. Parameters included a batch size of 32, training for 1 epoch, and a learning rate of 2e-5. Output attentions and hidden states were disabled to optimise computation. The model ran on available hardware, either GPU or CPU, with an AdamW optimiser and epsilon set at 1e-8.

A linear learning rate scheduler with zero warmup steps was integrated to adjust training pace. Gradient clipping capped at 1.0 prevented exploding gradients during backpropagation. Each training epoch involved a forward pass, loss calculation, and optimisation steps. The fine-tuned model was saved based on validation performance, ensuring it captured contextual patterns effectively for distinguishing text sources.

##### FocusCNN Implementation
FocusCNN was developed as a convolutional neural network for text classification. This architecture focused on capturing local patterns within text data through convolutional layers. Key parameters included kernel sizes for feature extraction and pooling layers to reduce dimensionality. Dense layers were added for final classification into two categories. The model was trained over multiple epochs with a batch size suited to hardware constraints, ensuring efficient processing of the large dataset.

Dropout layers were incorporated to prevent overfitting during training. The optimiser used was standard for deep learning tasks with a learning rate adjusted for stable convergence. Data was fed in batches, allowing the model to learn from sequential patterns in text. This setup aimed to detect structural differences between AI and human content, focusing on specific textual features for accurate categorisation.

##### AttentionCNN Implementation
AttentionCNN combined convolutional layers with attention mechanisms to prioritise relevant text segments. This model was built using a deep learning framework with layers designed to focus on contextual importance. Parameters included attention heads to weigh different parts of the input and convolutional filters for feature extraction. Batch size and epoch count were set to balance training time and performance on the dataset.

Training involved optimising loss through a suitable algorithm with a learning rate fine-tuned for gradual improvement. Pooling and normalisation layers ensured manageable data dimensions and stable gradients. This approach aimed to capture both local and global text patterns, enhancing the model's ability to differentiate between text origins. The structure was tested iteratively to refine its focus on key textual cues.

##### Context RF Implementation
Context RF was implemented as a traditional machine learning classifier for text categorisation. This random forest model was trained on engineered features derived from text data. Core parameters included the number of trees and depth limits to control complexity. Feature importance was inherently calculated to identify key predictors. The model processed data in batches to handle the large volume of records efficiently.

Training focused on minimising classification errors through ensemble techniques. Random sampling of features at each split ensured diversity in decision paths. The setup was straightforward, requiring minimal hyperparameter tuning compared to neural networks. This model aimed to provide a baseline for comparison, capturing statistical patterns in textual features to distinguish between AI and human-generated content effectively.

#### Explainable AI Approach
An explainable AI framework was integrated to interpret model decisions for text classification, focusing on DistilBERT as the primary model due to its superior performance. The method aimed to highlight which text segments influenced predictions most. Implementation involved post-hoc analysis tools to map input features to output labels, using attention weights inherent in the transformer architecture. No additional parameters were introduced beyond those used in model training.

The process extracted attention scores across layers, aggregating them to identify key tokens driving classification. This was applied to sample predictions across both AI and human text categories. Visualisations were generated to map these influences, aiding in understanding decision rationale. The approach was tested on validation data to ensure consistency, providing clarity on how specific words or phrases swayed the model towards a particular class during evaluation.

#### Obstacles and Resolutions
During the project on AI versus human text detection, several hurdles emerged, yet solutions were devised to address them. Handling the large dataset of 487235 records posed memory constraints, often slowing down processing. Optimising data loading in chunks mitigated this, ensuring smoother operations. Balancing the class distribution, with human text at 62.8% and AI at 37.2%, was another challenge. Sampling techniques were applied to even out representation, improving model fairness.

As per research by Mishra et al. (2020), text preprocessing inconsistencies can impact outcomes, which was observed with varied text formats. Standardising cleaning steps countered this issue effectively. Another concern was model interpretability, especially with complex architectures. According to Çetin and Yıldız (2022), transparent methods aid trust, so explainable tools were integrated to clarify decisions. These solutions maintained project progress, addressing technical barriers with practical adjustments.
